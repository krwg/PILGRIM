# Дело в Ницце: Пилигрим — сайт-читалка

Статический сайт без сборки: один `index.html`, главы — в `chapters/*.md`.
Работает на GitHub Pages как есть, никакого сервера не нужно.

## Структура

```
index.html                  ← точка входа (лендинг + читалка, роутинг по #хэшу)
assets/css/style.css        ← все стили и палитра
assets/js/app.js            ← вся логика: роутер, парсер глав, подсказки
chapters/manifest.json      ← список глав (для добавления НОВОЙ главы трогать только этот файл + .md)
chapters/ch01.md            ← глава 1 (рабочий пример)
chapters/ch02.md            ← глава 2 (рабочий пример)
chapters/_TEMPLATE.md       ← шаблон для новой главы, копировать и заполнять
```

## Как добавить новую главу — без правки кода

1. Скопируйте `chapters/_TEMPLATE.md` → `chapters/ch04.md` (или как удобно назвать файл).
2. Заполните `title`, `date`, `era`, эпиграф — и сам текст ниже.
3. Добавьте одну запись в `chapters/manifest.json`:

```json
{
  "id": "ch04",
  "number": 4,
  "title": "Bureau de Presse (Бюро печати)",
  "era": "1940 · Ницца",
  "status": "soon",
  "publish_date": "2026-07-03",
  "file": "chapters/ch04.md"
}
```

4. Закоммитьте оба файла. Сайт сам подхватит главу: она появится в «Содержимом дела»
   как запись с зачёркнутым/закрашенным названием и датой рассекречивания, а в день
   `publish_date` (если стоит `"status": "published"` или вы вручную смените статус)
   откроется по ссылке.

**Важный нюанс про анонсы.** GitHub Pages раздаёт публичный репозиторий — если
вы запушите `ch04.md` с полным текстом главы заранее и просто поставите
`"status": "soon"`, текст всё равно будет физически лежать в репозитории и
доступен по прямой ссылке или через историю коммитов. Если хотите анонсить
дату, но не палить текст раньше срока — добавляйте запись в `manifest.json`
сразу (заголовок, эпоха, дата), а сам файл `chapters/chNN.md` коммитьте
только в день публикации. До этого момента карточка главы в «Деле» будет
просто красиво показывать дату рассекречивания.

## Формат файла главы

Front-matter (вверху, между `---`):

```yaml
---
title: Название главы
date: Месяц год, Город
era: 1944 · Город
epigraph: |
  Текст эпиграфа, можно в несколько строк.
epigraph_source: Источник цитаты
---
```

Дальше — обычный текст. Правила минимальные:

- **Абзацы** разделяются пустой строкой — и всё.
- **Сцены** разделяются строкой из трёх звёздочек: `***`
- **Подсказки по терминам** (то, что вы называли «справкой на полях»):
  поставьте маркер сразу после слова — `кальвадоса[[4]]` — а определение
  допишите в любом месте файла строкой `[[4]]: Кальвадос — яблочный бренди…`.
  Удобнее всего складывать все определения внизу файла одним блоком, как
  сделано в `ch01.md` и `ch02.md`. При клике/тапе на термин на сайте
  появится всплывающая карточка с определением.
- **Документ-вставка** (досье, приказ, отчёт наблюдения) — оборачивается так:

  ```
  :::document
  ::label:: ДОКУМЕНТ 1 / EXHIBIT 417 GRS

  Текст документа абзацами, как обычно.

  ::margin:: Рукописная пометка на полях, если нужна.
  :::
  ```

  Получится визуально обособленная архивная вклейка — другой шрифт,
  лёгкий поворот, рамка слева цвета штемпеля.

## Локальная проверка перед публикацией

Файлы читаются через `fetch()`, поэтому открыть `index.html` двойным
кликом не получится (браузер блокирует fetch с `file://`). Поднимите
любой локальный сервер из папки сайта:

```
python3 -m http.server 8000
```

и откройте `http://localhost:8000`.

## Публикация на GitHub Pages

1. Залить содержимое этой папки в корень репозитория (или в ветку,
   которую вы укажете для Pages).
2. В настройках репозитория → Pages → Source: ветка `main`, папка `/ (root)`.
3. Сайт появится на `https://<ваш-логин>.github.io/<репозиторий>/`.

## Что можно докрутить дальше (по желанию)

- Разделить «дни» 1940 и 1944 визуально цветом полосы у `.chapter-head`
  (токены `--era-1940` / `--era-1944` уже заведены в CSS — сейчас они
  используются только в бейджах на лендинге).
- RSS/Atom-фид для анонсов глав, если захотите оповещать читателей не
  только через сам сайт.
- Страница «Альбом документов» — отдельный список всех `:::document`
  вставок по всей книге, чтобы читать досье отдельно от текста.
